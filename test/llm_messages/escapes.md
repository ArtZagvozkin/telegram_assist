## Демонстрация Экранирования Символов Markdown

Markdown — это мощный язык разметки, но иногда нам нужно отобразить сами символы, которые используются для форматирования, а не их функциональность. Для этого используется **обратный слеш** (`\`), который выполняет функцию экранирования.

Ниже приведены примеры экранирования наиболее часто используемых символов Markdown:

### 1. Экранирование Звёздочек (\*) и Подчеркиваний (\_)

Символы `*` и `_` обычно используются для выделения текста (курсив или жирный шрифт). Чтобы отобразить их как литералы, мы их экранируем:

*   Обычный курсив: *это курсив*
*   Экранированный звёздочки: Мы хотим напечатать это: `\*` и `\*`.
*   Обычный жирный шрифт: **это жирный шрифт**
*   Экранированное подчеркивание: Нам нужно увидеть символ `\_` без изменения форматирования.

### 2. Экранирование Символов Заголовка (\#)

Символ решетки (`#`) используется для обозначения заголовков. Чтобы отобразить его как обычный символ:

*   Обычный заголовок: `# Заголовок Уровня 1`
*   Экранированный символ: Чтобы показать символ хештега, используйте `\#`.

### 3. Экранирование Скобок для Ссылок (\[ \])

Квадратные скобки `[` и `]` используются для создания гиперссылок и для обозначения ссылок на изображения. Чтобы отобразить их буквально:

*   Обычная ссылка: [Текст ссылки](URL)
*   Экранированные скобки: Если вам нужно отобразить структуру: `\[Это просто текст\]`, а не ссылку.

### Сводная Таблица Экранирования

| Символ Markdown | Как выглядит экранирование | Результат (Отображение) |
| :-------------: | :------------------------: | :---------------------- |
| `*`             | `\*`                       | \*                      |
| `_`             | `\_`                       | \_                      |
| `#`             | `\#`                       | \#                      |
| `[`             | `\[`                       | \[                      |
| `]`             | `\]`                       | \]                      |

Использование обратного слеша (`\`) перед специальным символом заставляет Markdown-парсер интерпретировать его как обычный текстовый символ, игнорируя его форматирующие свойства.
